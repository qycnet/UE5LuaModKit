### 2. MOD 发布流程

1. **准备工作**
   - 完成 MOD 测试
   - 准备发布文档
   - 打包所需文件

2. **提交审核**
   - 提交 MOD 到仓库
   - 等待审核反馈
   - 根据反馈修改

3. **发布维护**
   - 发布 MOD
   - 收集用户反馈
   - 定期更新维护

1. **发现问题**
   - 记录文档问题
   - 确认问题范围
   - 提出改进建议

## 代码贡献

### 1. 代码提交规范

#### 提交消息格式
```
<类型>: <描述>

[可选的详细描述]

[可选的关闭问题引用]
```

类型包括：
- `feat`: 新功能
- `fix`: 错误修复
- `docs`: 文档更改
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

#### 代码风格
```lua
-- 命名规范
local function CalculateDamage(baseDamage, multiplier)
    -- 常量使用全大写
    local MIN_DAMAGE = 1
    
    -- 使用有意义的变量名
    local finalDamage = math.max(baseDamage * multiplier, MIN_DAMAGE)
    return finalDamage
end

-- 类/模块命名使用 PascalCase
local DamageCalculator = {
    -- 方法命名使用 camelCase
    calculateTotalDamage = function(self, ...)
        -- 实现
    end
}
```
